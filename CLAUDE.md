# 피파 전적관리 — 프로젝트 규칙

## 프로젝트 개요

넥슨 오픈API로 EA SPORTS FC 온라인 전적을 조회·집계하는 PyQt6 데스크톱 앱.

| 파일 | 역할 |
|------|------|
| `app_main.py` | PyQt6 UI — 검색 바, 요약 패널, 전적 표, 조회 워커 스레드(`MatchLoader`) |
| `nexon_api.py` | 넥슨 오픈API 클라이언트(`FCOnlineAPI`). **엔드포인트 경로·에러코드 상수가 전부 여기 상단에** |
| `models.py` | 매치 상세 JSON → `MatchSummary` 파싱, `Stats` 집계 |
| `config.py` | `.env`에서 API 키 로드, 매치 종류·조회 개수 기본값 |
| `check_api.py` | 터미널 연결 점검 — GUI 띄우기 전 키·엔드포인트 확인용 |

```powershell
python check_api.py <닉네임>   # API 점검
python app_main.py             # 앱 실행
```

필수 패키지: `pip install -r requirements.txt` (PyQt6, requests, python-dotenv)

## API 규칙 (이 프로젝트 고유 — 가장 중요)

1. **API 키는 절대 커밋하지 않는다.** 키는 `.env`에만 두고, 코드에서는 `config.API_KEY`로만 읽는다.
   `.env`는 `.gitignore`에 있다. 키를 코드·로그·커밋 메시지·이슈에 절대 쓰지 않는다.
   견본은 `.env.example`로 유지하고, 실제 키 대신 안내 문구만 넣는다.

2. **엔드포인트 경로는 `nexon_api.py` 상단 상수에만 둔다.** URL 문자열을 다른 파일이나
   함수 안에 흩뿌리지 않는다. 넥슨이 경로를 바꾸거나 내가 잘못 알았을 때 **한 곳만 고치면
   되게** 하기 위한 것 — 공식 문서가 JS 렌더링이라 자동 대조가 안 되므로 실제로 자주 겪는다.

3. **API 호출량을 아낀다.** 끝난 경기의 상세는 내용이 안 변하므로 `.cache/`에 디스크 캐시한다
   (`FCOnlineAPI.get_match_detail`). 새 조회 경로를 추가할 때도 캐시 우선을 먼저 검토한다.
   호출량 초과는 `OPENAPI00007`(429)로 돌아온다.

4. **응답 필드는 항상 `.get()`으로 방어적으로 읽는다.** 넥슨이 필드를 빼거나 이름을 바꿔도
   앱 전체가 죽으면 안 된다. 한 경기 파싱이 실패해도 나머지 조회는 계속돼야 한다
   (`MatchLoader._safe_detail` 패턴).

5. **API 에러는 사람이 읽는 말로 바꿔서 보여준다.** `nexon_api.ERROR_MESSAGES`에 코드별 한글
   메시지를 둔다. 사용자에게 `OPENAPI00005`를 그대로 띄우지 말 것 — "API 키가 유효하지 않습니다"로.

6. **새 엔드포인트를 붙이면 `check_api.py`에도 한 줄 추가한다.** GUI를 띄우지 않고 터미널에서
   `[OK]`/`[FAIL]`로 확인할 수 있어야 한다.

## PyQt6 규칙 (실수 방지 — 실제로 당한 것들)

1. **콤보박스는 항상 `NoScrollComboBox`.** `QComboBox` 직접 사용 금지 — 스크롤 휠이 지나가면
   값이 의도치 않게 바뀐다.

2. **새 위젯 클래스를 쓰면 `app_main.py` 상단 import를 확인한다.** 누락 시 실행 즉시
   NameError로 창이 그냥 사라져서, 원인 화면조차 안 뜬다.

3. **API 호출은 절대 UI 스레드에서 하지 않는다.** 네트워크 대기가 창을 얼린다.
   전부 `QThread` 워커(`MatchLoader`)에서 돌리고 시그널로 결과를 넘긴다.

4. **창을 닫을 때 워커를 정리한다.** `closeEvent`에서 `cancel()` → `wait()`.
   안 하면 백그라운드 스레드가 살아남아 종료가 안 되거나 에러창이 뜬다.

5. **재할당되는 전역은 이름 import 금지 — 모듈 경유로 읽는다.**
   `global X`로 재대입되는 전역을 `from mod import X` 하면 **import 시점 값이 복사**돼
   재할당이 안 보인다. 문법 검사·테스트가 전부 통과하는데 **기능만 조용히 죽는다.**
   옮기기 전에 `grep "global X"`로 재할당 여부를 먼저 확인할 것.
   (다른 프로젝트에서 두 번 당한 함정 — 언어 차원이라 프로젝트를 안 가린다.)

## 작업 방식

### 난이도별

| 난이도 | 방식 |
|--------|------|
| 낮음 (버그픽스·UI 조정·필드 추가) | 설계 없이 바로 구현 |
| 중간 이상 (새 화면·DB 도입·구조 변경) | **설계 먼저 → 합의 후 구현** |

설계에 포함할 것: 수정할 파일 목록 · 핵심 변경(함수명·반환값) · 의존성·주의사항.

### Git

- **기능 단위로 완성 후 커밋한다.** 작업 중간 상태는 커밋하지 않는다.
- 커밋 후 **push까지 자동으로** 진행한다(원격이 붙어 있으면).
- `.env`, `.cache/`, `__pycache__/`, `dist/`, `build/`는 커밋하지 않는다(`.gitignore`).

### 검증

- **코드를 고쳤으면 실제로 돌려본다.** `python check_api.py <닉네임>`이 기본 스모크.
- 파싱 로직을 고쳤으면 **가짜 응답 dict로 파싱 경로를 먼저 확인**한다(네트워크 없이 즉시 검증).
- 진행 상황·시간은 **추측해서 보고하지 않는다.** 실제 출력·시계로 확인한 것만 말한다.

### 문서

- **README.md를 최신으로 유지한다.** 지금은 유일한 사용자 문서다.
  파일이 늘거나 실행 방법·엔드포인트가 바뀌면 그 자리에서 갱신.
- 코드 주석은 "왜 이렇게 했는지"가 비직관적일 때만 단다. 코드가 이미 말하는 걸 반복하지 않는다.
